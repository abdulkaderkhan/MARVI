Well Optimization Process

The current goal of the MARVI Project is to determine the optimal number of wells within the network to be monitored. This step would be done by utilizing the Ramer-Douglas-Peucker algorithm.

So, what is the Ramer-Douglas–Peucker algorithm?

This algorithm is used to smooth and reduce the number of points in an estimated curve. Say you have a set of points in a roughly linear fashion, for example. This algorithm takes the first and last points in this estimated ‘line,’ and then trims off the points that are too far away from the start and end. Whether or not a point in the middle of this line is trimmed depends on a chosen distance ‘epsilon.’ A visual of this concept can be seen below:



What does this algorithm have to do with MARVI?

In RStudio, a line was constructed using all of the wells (i.e. points) in the well network. The resulting line is a bit chaotic and has plenty of likely unecessary points. The algorithm will help trim anything extra and result in a much cleaner shape.


Applying the algorithm

Once the points are in RStudio, the algorithm can be applied and it will turn that former mess of points into a nice scatterplot. Next to the scatterplot are the resulting coordinates of the remaining points.

Finally, it was necessary to test the watershed using kriging interpolation with the new well network.

Kriging

The Kriging method is a method of spatial interpolation, where acquired samples are used to approximate and predict at unsampled locations. This implies that Kriging can be used to predict the best wells to use now that we have a smaller list, thanks to the earlier algorithm.
